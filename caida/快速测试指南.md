# 购物车功能 - 快速测试指南

## 📋 测试前准备

### 1. 安装MySQL数据库
确保您的计算机已安装MySQL 8.0或更高版本。

### 2. 初始化数据库
在命令行中执行以下命令：

```bash
# 进入项目目录
cd D:\itfan\caida

# 执行数据库初始化脚本（需要输入MySQL密码）
mysql -u root -p < init_database.sql
```

或者使用MySQL Workbench等图形化工具执行 `init_database.sql` 文件。

### 3. 配置数据库连接
编辑文件：`src/main/resources/application.properties`

修改以下配置为您的实际数据库信息：
```properties
spring.datasource.username=root
spring.datasource.password=你的MySQL密码
```

## 🚀 启动步骤

### 方式一：使用批处理文件（推荐）
双击运行：`启动后端.bat`

### 方式二：使用Maven命令
```bash
cd D:\itfan\caida
mvn spring-boot:run
```

等待看到以下提示表示启动成功：
```
Started CaidaApplication in X.XXX seconds
```

## 🧪 测试方法

### 方法一：使用API测试页面（最简单）

1. 确保后端服务已启动
2. 在浏览器中打开：`test-api.html`
3. 依次测试各个API接口：
   - ✅ 添加商品到购物车
   - ✅ 获取购物车列表
   - ✅ 更新商品数量
   - ✅ 删除购物车商品
   - ✅ 获取购物车数量

### 方法二：使用前端页面

1. 确保后端服务已启动
2. 在浏览器中打开：`caida-qianduan/index.html`
3. 操作步骤：
   - 浏览商品列表
   - 点击"加入购物车"按钮
   - 点击右上角"购物车"查看
   - 调整商品数量
   - 删除商品
   - 查看总价

### 方法三：使用Postman或cURL

#### 1. 添加商品到购物车
```bash
curl -X POST http://localhost:8080/api/cart/add \
  -H "Content-Type: application/json" \
  -d "{\"userId\":1,\"productId\":1,\"quantity\":2}"
```

#### 2. 获取购物车列表
```bash
curl http://localhost:8080/api/cart?userId=1
```

#### 3. 更新商品数量
```bash
curl -X PUT http://localhost:8080/api/cart/update \
  -H "Content-Type: application/json" \
  -d "{\"cartId\":1,\"quantity\":3}"
```

#### 4. 删除购物车商品
```bash
curl -X DELETE http://localhost:8080/api/cart/remove/1
```

#### 5. 获取购物车数量
```bash
curl http://localhost:8080/api/cart/count?userId=1
```

## 🔍 常见问题排查

### 问题1：无法连接到数据库
**症状**: 启动时出现数据库连接错误

**解决方案**:
1. 检查MySQL服务是否启动
2. 验证用户名和密码是否正确
3. 确认数据库 `caida_db` 已创建
4. 检查端口3306是否被占用

### 问题2：端口8080被占用
**症状**: 启动失败，提示端口被占用

**解决方案**:
在 `application.properties` 中添加：
```properties
server.port=8081
```
然后修改前端 `script.js` 中的API地址为 `http://localhost:8081/api`

### 问题3：跨域错误
**症状**: 前端调用API时出现CORS错误

**解决方案**:
后端已配置 `@CrossOrigin(origins = "*")`，如仍有问题：
1. 检查浏览器控制台具体错误信息
2. 确保后端服务正常启动
3. 尝试清除浏览器缓存

### 问题4：商品图片不显示
**症状**: 购物车中商品图片无法加载

**解决方案**:
1. 确认图片存在于 `caida-qianduan/images/` 目录
2. 检查图片文件名是否正确
3. 确保使用相对路径访问
